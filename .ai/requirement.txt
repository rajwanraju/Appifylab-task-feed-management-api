# Social Feed Backend Assignment (Express.js + PostgreSQL)

## Objective

Build a production-ready REST API for a social feed application.

The frontend is a separate Next.js application and will consume this API.

The goal is to implement only the features defined in the assignment while keeping the code clean, scalable, and easy to maintain.

Do not overengineer the solution.

---

# Tech Stack

* Node.js 22+
* Express.js
* TypeScript
* PostgreSQL
* Prisma ORM
* Redis (optional caching)
* JWT Authentication
* Multer for file uploads
* Docker & Docker Compose
* Zod or Joi for validation
* Pino for logging
* Jest + Supertest for testing

---

# Non-Functional Requirements

The system should be designed assuming:

* Millions of posts and reads
* High concurrency
* Horizontal scalability
* Stateless API design
* Good database indexing practices
* Security and UX are top priorities

Do not implement microservices, Kafka, CQRS, or Kubernetes.

Keep the architecture simple and production-ready.

---

# Authentication

Use JWT Authentication.

## Register

Fields:

* firstName
* lastName
* email
* password

Requirements:

* email must be unique
* hash password using bcrypt
* validation
* return user and token

---

## Login

Fields:

* email
* password

Requirements:

* validate credentials
* return JWT token
* return authenticated user

---

## Logout

JWT is stateless.

Simple frontend token removal is sufficient.

Optional:

Implement token blacklist using Redis.

# Feed Pagination

Implement cursor pagination.

Avoid offset pagination.

Example:

```http
GET /api/posts?cursor=123&limit=20
```

---

# Database Requirements

Use PostgreSQL.

Design proper relationships and foreign keys.

Create indexes for:

```sql
posts(created_at DESC)
posts(user_id)
posts(visibility)
comments(post_id)
replies(comment_id)
likes(user_id, target_type, target_id)
```

Create unique constraint:

```sql
(user_id, target_type, target_id)
```

to prevent duplicate likes.

---

# Image Upload

Use Multer.

Store files locally:

```text
/uploads
```

Image upload is only required for posts.

No image processing is required.
